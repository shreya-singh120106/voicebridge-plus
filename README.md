## рҹҢҚ The Problem

Over **400 million people** in India alone struggle to access essential services вҖ” healthcare, banking, government schemes вҖ” due to:

- рҹ“ө Low digital literacy and complex app interfaces
- рҹҢҗ Language barriers (Hindi, regional languages vs English-only UIs)
- рҹ“Ӣ Multi-step processes that require reading & typing
- рҹ§© No memory вҖ” having to repeat the same information every single time

**People miss doctor appointments. They miss scholarship deadlines. They don't know their account balance.**

Not because the services don't exist вҖ” but because the **interface is broken for them.**

---

## рҹ’Ў The Solution

**VoiceBridge+** is a voice-first AI agent that lets anyone interact with essential services through **natural conversation** вҖ” in Hindi, English, or Hinglish.

No forms. No menus. No typing. Just **talk.**

```
User:  "Doctor appointment chahiye"
AI:    "Bilkul! Aap kis city mein hain?"
User:  "Mumbai"
AI:    "Kaun se specialist chahiye?"
User:  "Heart doctor"
AI:    "вң… Dr. Rajesh Kumar вҖ” Apollo Hospital, Mumbai | Token: APL-4821"
```

The AI **remembers** you, **speaks your language**, and **completes tasks** вҖ” not just answers questions.

---

## вңЁ Features

### рҹҺҷпёҸ Voice Interaction
- One-tap mic button to start/stop recording
- Real-time speech-to-text via Web Speech API
- Optional text-to-speech for AI responses
- Live voice waveform animation while listening
- Supports Chrome, Safari, Edge, and Android browsers

### рҹӨ– Agent Action Mode *(The WOW Feature)*
The AI doesn't just answer вҖ” it **completes tasks** step by step.

| Task | Steps | What Happens |
|------|-------|--------------|
| рҹҸҘ Doctor Appointment | 4 steps | City вҶ’ Specialist вҶ’ Time Slot вҶ’ Confirmed Booking |
| рҹҸҰ Bank Balance | 3 steps | Bank Name вҶ’ Account Digits вҶ’ Balance Retrieved |
| рҹҺ“ Scholarship Guidance | 4 steps | Level вҶ’ Category вҶ’ Income вҶ’ Matched Scholarships |

Each flow maintains full **session state** вҖ” users never need to repeat themselves mid-conversation.

### рҹ§  Memory & Personalization *(Powered by Vector DB)*
- Past interactions stored as semantic embeddings (Qdrant / in-memory)
- Similarity search retrieves relevant context before every response
- Returning users get **instant recognition** вҖ” skipping steps they've done before

```
# First visit
User: "Book a doctor appointment"
AI:   вҶ’ Asks city, specialist, time (full flow, 4 steps)

# Return visit  
User: "Same doctor book kar do"
AI:   вҶ’ "I remember you! Dr. Sharma, Apollo Mumbai вҖ” confirm? вң…" (1 step)
```

### рҹҢҗ Multilingual Intelligence
- Auto-detects Hindi, English, and Hinglish (code-switching)
- Responds in the **same language and style** the user uses
- Supports Devanagari script rendering in UI
- No language selection needed вҖ” just talk naturally

### рҹ’¬ Conversational Intelligence
- Intent detection: task / query / help / recall
- Handles vague or incomplete inputs gracefully
- Multi-turn context across the full conversation
- Warm, accessible tone вҖ” designed for low-literacy users

### рҹҺӯ Demo Mode
- Pre-loaded demo user "Priya" with rich memory history
- Instant showcase of returning user personalization
- 5 curated demo scenarios for live presentations

### Data Flow
1. **User speaks** вҶ’ Web Speech API transcribes to text
2. **Text sent** to `POST /api/chat` with `sessionId` + `userId`
3. **Language detected** (Hindi / English / Hinglish)
4. **Intent classified** (doctor / bank / scholarship / greet / recall)
5. **Memory retrieved** via vector similarity search (top 3 relevant memories)
6. **Task state machine** advances the conversation flow
7. **LLM generates** context-aware response (OpenAI or rich mock)
8. **Response stored** as embedding for future recall
9. **Frontend displays** response + optional TTS playback

## рҹҡҖ Setup Instructions

### Prerequisites

- Node.js 18+
- npm or yarn
- (Optional) OpenAI API key for real AI responses
- (Optional) Qdrant instance for persistent vector memory

### 1. Clone the repository

```bash
git clone https://github.com/shreya-singh120106/voicebridge-plus.git
cd voicebridge-plus
```

### 2. Set up the Backend

```bash
cd backend
npm install

# Copy and configure environment variables
cp .env.example .env
# Edit .env вҖ” OpenAI key is optional, app works without it
```

```bash
# Start the backend server
npm run dev
# Server runs at http://localhost:5000
```

### 3. Set up the Frontend

```bash
# In a new terminal
cd frontend
npm install

# Copy environment file
cp .env.example .env.local
# NEXT_PUBLIC_API_URL=http://localhost:5000 (default, no changes needed)
```

```bash
# Start the frontend
npm run dev
# App runs at http://localhost:3000
```

### 4. Open and explore

```
http://localhost:3000
```

That's it. No database setup, no API keys required to see the full demo. рҹҺү

---

### Optional: Real OpenAI Integration

Edit `backend/.env`:

```env
OPENAI_API_KEY=sk-your-key-here
```

Restart the backend. All responses now use GPT-4o-mini with full context awareness.

### Optional: Real Qdrant Integration

```bash
# Run Qdrant locally via Docker
docker run -p 6333:6333 qdrant/qdrant
```

Edit `backend/.env`:
```env
QDRANT_URL=http://localhost:6333
```

Then update `vectorMemory.js` to use the `@qdrant/js-client-rest` package вҖ” the service interface is identical, making the swap a drop-in replacement.
